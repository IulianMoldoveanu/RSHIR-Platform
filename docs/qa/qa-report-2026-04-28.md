## Report-only findings (prioritized)

### [P0] PR #8 — wrong `payment_status` filter in `markOrderPaidAndDispatch`
- **Where:** `apps/restaurant-web/src/app/api/checkout/order-finalize.ts:43` (PR #8 branch `chore/security-audit-2026-04-27`)
- **Symptom:** The new atomic race-guard filters `.eq('payment_status', 'PENDING')` but the schema uses `'UNPAID'` (values: `UNPAID/PAID/REFUNDED/FAILED`). The UPDATE clause never matches, so the loser-side check never fires — but more critically, **the first caller also never marks the order PAID**, meaning all card orders remain in `UNPAID` state indefinitely after a successful Stripe payment. Stripe webhooks and `/confirm` both call this function; both return without dispatching.
- **Suggested fix:** Change `.eq('payment_status', 'PENDING')` to `.eq('payment_status', 'UNPAID')` on line 43.
- **Why not auto-fixed:** `order-finalize.ts` is in the protected `api/checkout/**` path (payment-math guardrail).
- **Action required:** Fix before merging PR #8. ⚠️ Do not merge #8 as-is — card orders will appear to succeed on the client but never be recorded as paid.

### [P1] PR #9 — `resolveTenantFromHost` silently fails when `NEXT_PUBLIC_PRIMARY_DOMAIN` unset
- **Where:** `apps/restaurant-web/src/lib/tenant.ts:77` (PR #9 branch `chore/configurable-primary-domain`)
- **Symptom:** When `NEXT_PUBLIC_PRIMARY_DOMAIN` is not set, `SUBDOMAIN_BASES` contains only `['lvh.me']`. Slug-routed tenant traffic (`<slug>.<apex>`) falls through the subdomain check and hits the custom-domain lookup, returning no tenant → every storefront page returns the "restaurant not found" error. Silent production outage on a misconfigured deploy with no diagnostic log.
- **Suggested fix:** Add `console.error` + early return, or add a `process.env.NODE_ENV === 'development'` fallback to a default apex.
- **Why not auto-fixed:** Human pushed to branch within 4 h.

### [P1] PR #9 — `NOMINATIM_CONTACT_EMAIL` inaccessible in browser bundle
- **Where:** `apps/restaurant-web/src/lib/zones/nominatim.ts:22` (PR #9 branch)
- **Symptom:** `geocodeAddressRo` is consumed by `CheckoutClient` (`'use client'`). `process.env.NOMINATIM_CONTACT_EMAIL` is not `NEXT_PUBLIC_*` so it's `undefined` in the browser, always falling back to `ops@example.com`. The env-driven configuration added in PR #9 is silently ineffective.
- **Suggested fix:** Rename to `NEXT_PUBLIC_NOMINATIM_CONTACT_EMAIL` in code and Vercel project settings.
- **Why not auto-fixed:** Human pushed to branch within 4 h (env var rename requires coordinated Vercel settings update).
