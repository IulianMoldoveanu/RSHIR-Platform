## Report-only findings (prioritized)

### [P1] PR #481 — test-mode webhook secret not wired (Codex: r3251602373)

- **Where:** `apps/restaurant-web/src/app/api/webhooks/stripe/route.ts:39,52` (referenced by Codex on PR #481)
- **Symptom:** When `paymentSurface.mode === 'card_test'`, Stripe Checkout Sessions are created with the TEST key, but the webhook handler only verifies with `STRIPE_WEBHOOK_SECRET` (the live endpoint secret). Test-mode `payment_intent.succeeded` events are rejected as invalid signatures, so demo card payments complete at Stripe but the order is never marked PAID or dispatched.
- **Suggested fix:** Add `STRIPE_WEBHOOK_SECRET_TEST` env var and a mode-aware selection in the webhook route (or configure a separate test endpoint in the Stripe dashboard). Requires human review — touches `apps/restaurant-web/src/app/api/webhooks/stripe/**` (protected Stripe path per QA charter).
- **Why not auto-fixed:** Protected Stripe/payment path — requires human sign-off.

### [P1] PR #472 — migration ordering: trust_auto_promotion ALTER before CREATE (Codex: r3251529486)

- **Where:** `supabase/migrations/20260514_001_trust_auto_promotion.sql:23`
- **Symptom:** On a fresh database, migrations run in filename order. `20260514_001` reaches `ALTER TABLE public.tenant_agent_trust` before `20260608_002_ai_master_orchestrator.sql` creates that table. Clean Supabase rebuild fails with `relation "public.tenant_agent_trust" does not exist`.
- **Suggested fix:** Rename this migration after `20260608_002` (e.g. `20260608_003` or a later slot) or add a `CREATE TABLE IF NOT EXISTS` guard. Requires human review — touches `supabase/migrations/**`.
- **Why not auto-fixed:** Migrations are read-only for QA agent.

### [P1] PR #472 — trust promotion stats are not category-scoped (Codex: r3251529490)

- **Where:** `supabase/functions/trust-promote-daily/index.ts:155`
- **Symptom:** The window lookup uses `restaurant_id|agent_name` as the key, omitting `action_category`. Clean runs in one category (e.g. `marketing.social.draft`) can auto-promote a different category for the same agent (e.g. `marketing.social.publish`) that has never had clean executions. A revert in one category also resets/demotes all categories for that agent.
- **Suggested fix:** Include `action_category` in the window key and the view query join. Requires human review — outside restaurant-web/admin write scope.
- **Why not auto-fixed:** supabase/functions is outside write scope.

### [P1] PR #475 — embedding backfill void promise not registered with EdgeRuntime.waitUntil (Codex: r3251537882)

- **Where:** `supabase/functions/_shared/master-orchestrator.ts:406`
- **Symptom:** `void backfillRunEmbedding(...)` is not registered with `EdgeRuntime.waitUntil`. After the HTTP handler sends the dispatch response, the worker can be torn down before the OpenAI embed fetch and `copilot_agent_runs` update complete. Result: `embedding` column remains null, `match_agent_runs` has no history to retrieve.
- **Suggested fix:** Wrap with `EdgeRuntime.waitUntil(backfillRunEmbedding(...))` as done in `voice-incoming/index.ts:37` and `telegram-command-intake/index.ts:73`. Requires human review — outside restaurant-web/admin write scope.
- **Why not auto-fixed:** supabase/functions is outside write scope.
