### [P2] menu-imports TTL cron тАФ SQL delete leaks orphaned storage files
- **Where:** `supabase/migrations/20260508_010_menu_imports_ttl_fix.sql:44` (Codex on PR #372)
- **Symptom:** The TTL sweep deletes rows from `storage.objects` via raw SQL. Supabase storage treats these rows as metadata only тАФ the underlying file in the storage bucket is NOT removed, so the cron appears to succeed while leaking orphaned files and billing.

- **Why not auto-fixed:** Migration files are READ-ONLY in QA scope. Architecture change required.
